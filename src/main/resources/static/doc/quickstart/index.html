To run a test on the AQA platform a machine and RTPLAN must first be created ( account required, contact Jim Irrer or Natalie Baughan ) . This is only done once, and subsequently tests may be run against the machine many times by repeating steps 3 and 4.

a. Each institution will receive user accounts that will have access to run files and create additional accounts. This account must be created by, please contact them to get set up. They will send you your account details.

1: Create Machine

Administration → Machines → Create new Machine →

Fill fields to reflect your machine, and click Create , which will return to the list of machines.

2: Create RTPLAN

Click the name of the newly created machine, then the "Customize Plan" button at the bottom.

The Patient ID , Patient Name , and Plan Name are arbitrary, but can be helpful for tracking the machine's test activity.

The Tolerance Table Name should be one that exists in your treatment planning system. The Machine Name is given a default, but may need to be changed to match the name expected by the treatment machine.

Click Create to create the RTPLAN. This will bring up another page with a link to download the just-created RTPLAN.

3: Treat With the RTPLAN

  • Import the RTPLAN into your treatment system.

  • Verify the couch coordinates are correct for your machine. The couch needs to be retracted so it won't be on imager, but still in a position to beam on.

  • Schedule sessions and add integrated image template in scheduling. The plan needs to run as integrated, not continuous.

  • Imager should be at 1500 mm (not at ISO)

  • Schedule at machine, deliver in QA mode, and export the RTIMAGE files.

  • Manually enter the imager coordinates: 50cm, 0, 0

4: Run the Test

  • On the AQA site, click Run Procedure → Phase2

  • Drag and drop the RTIMAGE files into the upload box.

  • Click the Run button to start the analysis, which takes less than 2 minutes.

  • Results will be displayed automatically, showing an overview of results. Click on the tests for more details.
