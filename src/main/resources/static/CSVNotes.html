Notes on CSV Files

CSV files are intended as a way to provide access to the aggregated data for study and research. The data spans all institutions and is anonymized. Users can view the original values for their own institutions.

These files are generated manually by the system administrator. It is expected that they will be re-generated on a regular basis so as to include the latest data.

Data within each CSV is grouped by machine, with a separating blank row between machines. Within each machine, data is sorted by date. Maintenance records are embedded chronologically in the data.

The CSV files were designed with a bias towards being complete rather than minimal. It is expected that the user may discard or hide a majority of the columns to focus on what is pertinent. DICOM values have been provided to provide greater insight and possibly explain outliers.

Except for Maintenance Records, the CSV columns are organized in four main sections:

  • Institution, machine, and time stamps.
  • Data unique to the given type of analysis.
  • Machine characteristics, URL referencing report, and user that uploaded the data.
  • DICOM values associated with the data. This can be useful to find correlations between data characteristics and machine settings. Some data sets (Collimator Centering, VMAT, and Wedge) have two sets of DICOM files, in which case the columns names will be slightly different for each. The types of DICOM metadata are the same for all CSV files. The column descriptions for DICOM fields are appended with DICOM GGGG,EEEE to indicate the standard group and element as hex characters. Since the same DICOM files can be used for multiple tests, the same values may appear in different CSV files.

Also note that CSV files are one way to access data, but it is also possible to access the database directly. This allows the use of tools such as Tableau or PowerBI to make your own custom reports. For arranging direct access to the database, contact the site administrator.
